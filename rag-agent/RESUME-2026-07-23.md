## 재개할 일 (우선순위 순)

### 1. RealHiTBench 답변정확도 런 (지금 이 컴퓨터에서 돌아가는 중)
- 백그라운드로 `llama-3.3-70b-versatile` 실행 중. 컴퓨터 꺼도 됨 — 기록은
  `results/realhitbench_answer_accuracy_records.jsonl`에 쿼리별 저장되고 `--resume`이 이어감.
- 모집단: 집계질문 334개 중 층화 샘플 **100개 (seed=0, 결정적 — 항상 같은 100개)**.
- 70b TPD 100k → 하루 ~30개. 매일 아래 명령 재실행 (429로 끊기면 자동 저장됨):
```bash
cd ~/T2/rag-agent
GROQ_API_KEY=<키> PYTHONPATH=. python3 scripts/realhitbench_answer_accuracy.py \
  --solver-model llama-3.3-70b-versatile --codegen-max-tokens 160 \
  --sample 100 --seed 0 --injected-first --resume
```
- 완료 조건: records 100줄. 진행 확인: `wc -l results/realhitbench_answer_accuracy_records.jsonl`
- 결과: `results/realhitbench_answer_accuracy.json` (strict EM base/treat + McNemar + CompStrucCata별)

### 2. HiTab gpt-oss-120b 주입 런 완주 (92→161)
- gpt-oss-120b TPD(200k)는 **내일 저녁(~19시)쯤 리셋**. 리셋 후:
```bash
cd ~/T2/rag-agent
GROQ_API_KEY=<키> PYTHONPATH=. python3 scripts/answer_accuracy_injection.py \
  --solver-model openai/gpt-oss-120b --codegen-max-tokens 1024 --flips-first --resume \
  --records results/answer_accuracy_injection_gptoss120b_v2_records.jsonl \
  --out results/answer_accuracy_injection_gptoss120b_v2.json
```
- 완료 조건: records 161줄. 남은 69개 ≈ 300k 토큰 ≈ 이틀 치.
- **주의**: 완주하면 헤드라인 Δ+0.087은 +0.05 근처로 희석될 예정(flips-first 편향 해소).
  p는 유지 전망. 보고 때 "n=92 시점"이라고 명시할 것.
